# Internal Copilot Runbook

このファイルは、内部環境のGPT-5.5 Thinkingに渡すための実行手順書です。

外部環境で作成した公表済み審査報告書コーパスを参照しつつ、企業からの申請資料、未公表データ、内部判断は内部環境でのみ扱う。

## Mission

今回の目的は、脳血管イベントに対する抗凝固薬について、申請資料から品目固有の事実と審査論点を抽出し、公表済みPMDA審査報告書の書きぶりを参考に、審査報告書ドラフトを作成することである。

## Non-Negotiable Rules

1. 申請資料に明示されている事実と、審査上の推論を分ける。
2. 公表済み報告書から、今回品目の事実を補完しない。
3. 古い報告書は論点抽出に使い、文体は最近報告書に寄せる。
4. 出血、頭蓋内出血、抗血小板薬併用、腎機能、肝機能、高齢者、日本人データ、用量設定は必ず確認する。
5. 根拠が見つからない事項は、ドラフト本文に混ぜず「不足情報・要確認事項」に置く。
6. 参照した公表報告書は、品目名と参照した論点を明示する。
7. 最終ドラフトでは、承認可否の結論、留保、製造販売後の情報収集を混同しない。
8. 比較対照薬に対する非劣性が検証されていない場合は、非劣性未検証を明記した上で、対象集団の限定、代替根拠、添付文書注意喚起、製造販売後情報収集を整理する。

## Step 6: 過去報告書との対応表

内部資料から抽出した論点を、公表報告書のどの論点と対応させるか整理する。

```md
| 今回品目の論点 | 参照報告書 | 参照理由 | 使うもの | 使わないもの |
|---|---|---|---|---|
| 出血 | リクシアナ 2025 | 抗凝固薬で出血評価が近い | 評価構造 | 疾患固有の事実 |
| 頭蓋内出血 | エフィエント 2014 | 抗血小板薬で脳血管既往が論点 | 注意喚起の観点 | 用量そのもの |
| 非劣性未検証 | エフィエント 2021 | 比較対照薬に対する非劣性未検証でも対象集団を限定 | 論理構成、留保、製販後接続 | クロピドグレル固有の事実 |
| 固定用量・切替え | ユバンシ 2024 | 配合剤と切替えの整理が明確 | 忍容性確認、対象患者整理 | PAH固有の治療実態 |
```

## Step 7: 審査報告書ドラフト作成

ドラフトは次の順で作る。

1. 品目概要
2. 臨床的位置づけ
3. 有効性
4. 安全性
5. 用法・用量
6. 日本人データ
7. 製造販売後の検討事項
8. 総合評価

総合評価の基本型:

```md
提出された資料から、本品目の［対象疾患/対象患者］に対する［有効性の評価］は［示された/示唆された/一定の評価は可能］と判断する。

安全性については、［主要なリスク］に留意する必要があるものの、［試験成績、背景因子別検討、リスク管理策］を踏まえると、認められたベネフィットを踏まえて臨床的に許容可能と判断する。

ただし、［残る不確実性］については、［製造販売後調査/RMP/添付文書注意喚起］により情報収集又は注意喚起を行う必要がある。
```

## Step 8: 品質チェック

ドラフト作成後、次を確認する。

- 申請資料に根拠がない事実を書いていないか。
- 公表報告書の疾患固有事実を今回品目に混入していないか。
- 古い報告書の文体をそのまま使っていないか。
- 出血、頭蓋内出血、併用薬、腎機能、高齢者、日本人データが抜けていないか。
- 「有効性は示された」と「有効性は示唆された」の温度感が適切か。
- 「安全性は許容可能」と「製造販売後でさらに検討が必要」が矛盾なく接続されているか。
- 結論とRMP/添付文書/製造販売後調査の対応が取れているか。

## First Prompt To Run Internally

```md
以下の申請資料を読み、脳血管イベントに対する抗凝固薬の審査報告書作成に必要な論点を抽出してください。

公表済み審査報告書は、事実の補完ではなく、論点抽出と書きぶりの参考としてのみ使用してください。

特に、出血、頭蓋内出血、抗血小板薬併用、腎機能、肝機能、高齢者、低体重、日本人データ、用量設定、比較対照薬に対する非劣性検証の成否、製造販売後調査/RMPを必ず確認してください。

出力は以下の構造にしてください。

1. 品目概要
2. 主要な審査論点
3. 有効性論点
4. 安全性論点
5. 用法・用量論点
6. 日本人データ・外挿性
7. 参照すべき公表済み審査報告書と理由
8. 審査報告書ドラフトに使える書きぶり
9. 不足情報・要確認事項

各項目では、申請資料に記載された事実、審査上の推論、不足情報を分けてください。
```
