# Section 7 Clinical Data Draft Prompt

このプロンプトは、PMDA審査報告書の「7. 臨床的有効性及び臨床的安全性に関する資料並びに機構における審査の概略」を作成するために使用する。

公表済み審査報告書は、章立て、表の作り方、結果の書き方、留保の置き方の参考にのみ使う。今回品目の事実は、申請資料、CTD、試験総括報告書、照会回答等の内部資料に根拠があるものだけを記載する。

## Task

添付された申請資料から、審査報告書7項のドラフトを作成してください。

最初から審査判断だけを書かず、まず臨床試験データを審査報告書の形式で整理してください。各試験について、目的、試験デザイン、対象、主要な選択基準、用法・用量、解析対象、患者背景、有効性成績、安全性成績、中止例、死亡、重篤な有害事象、投与中止に至った有害事象を記載してください。

## Data Extraction Rules

1. 試験名、CTD番号、実施期間、実施地域、施設数、相、試験デザインを必ず抽出する。
2. 登録例数、無作為化例数、FAS、安全性解析対象集団、PPS等を分ける。
3. 主要評価項目は、定義、評価時点、解析方法、欠測値補完、非劣性限界値又は優越性判定基準を記載する。
4. 有効性結果は、群ごとの値、群間差、リスク比/ハザード比、95%CI、p値、非劣性/優越性の成否を可能な限り表で示す。
5. 安全性結果は、全有害事象、死亡、死亡以外の重篤な有害事象、投与中止に至った有害事象、注目すべき有害事象を分ける。
6. 抗凝固薬・抗血小板薬では、出血、大出血、臨床的に重要な非大出血、頭蓋内出血、消化管出血、脳梗塞/TIA、併用抗血栓薬、NSAIDs、腎機能、肝機能、高齢者、低体重を必ず拾う。
7. 日本人集団がある場合は、全体集団とは別に有効性及び安全性を整理する。症例数が少ない場合は精度の限界を明示する。
8. 非劣性が検証されなかった場合は、検証されなかった事実、追加試験/部分集団/作用機序/臨床的位置付けで補える範囲、補えない範囲を分ける。
9. 申請資料に見つからない数値は推測しない。`要確認` として残す。
10. 公表済み審査報告書の疾患固有事実、試験番号、数値を今回品目の事実として混入しない。

## Reference Style

7項本体の型として特に参照する。

- ユバンシ2024: 7項冒頭、主な臨床試験の概略表、第I相試験、第III相試験、主要評価項目表、安全性表、日本人集団の書き方。
- リクシアナ2025: 1試験中心の場合の7項構成、前治療、主要評価項目、出血リスク、追加調査なしの整理。
- エフィエント2021: 複数試験、用量設定、第III相試験、非劣性未検証、CYP2C19、低体重、高齢者、製造販売後調査への接続。
- イグザレルト2023: 小児/少数例、体重別用量、出血、RMP、製造販売後調査の書き方。

## Output Requirements

- まず7項本体のドラフトを作成する。
- その後に、`要確認事項` として不足データを箇条書きでまとめる。
- 判断文だけを先に書かず、試験データの整理を本文の中心に置く。
- 表はMarkdown表でよいが、後で審査報告書に移せる粒度にする。
